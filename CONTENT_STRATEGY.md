# Minpaku Resort 都市ページ 改善方針書

## 背景：なぜ改善が必要か

Google Search Consoleで都市ページが「重複コンテンツ」と判定され、インデックス登録に失敗している。

**根本原因：** 都市ページの約94%のHTMLが全都市で同一。Googleは「これは同じページが大量にある」と判断し、自分で"正規ページ"を選び直してしまう。

## 現状コンテンツ診断（Step 3〜5の根拠）

### 都市固有コンテンツの品質評価

| セクション | 状態 | 問題 |
|---|---|---|
| hero (tagline/copy) | ✅ 全都市ユニーク | なし |
| localRules (3枚) | ⚠️ 7/41都市が汎用テンプレ | karuizawa, nasu, kobe, aso, kagoshima, wakayama, takamatsu |
| marketAnalysis | ⚠️ 200〜350文字と浅い | 相場数字・国籍比率がない |
| cleaningFeature | ⚠️ 1〜2文のみ | kyoto/niseko/atamiは具体的、残りは薄い |
| cleaningPrices | ❌ 34/41都市で同一価格 | 地域差が全く反映されていない |
| localFaqs (3問) | ✅ ほぼ固有 | 問数が少ない |
| voices (お客様の声) | ❌ 39/41都市で同一フレーズ | 都市ごとに個別化が必要 |
| caseStudies stats | ❌ 全41都市で85%/4.9/140%UP | 全都市共通の数値は信頼性を損なう |
| staff | ❌ 31都市が田中・佐藤 | 全国に同一人物が存在するのは不自然 |
| numbers/strengths/problems | ❌ 全都市共通 | 都市ページに不要。トップ専用 |
| areas (全国タブ) | ❌ 全都市に全国マップ | 離脱を誘発する |

### Airbnbオーナー視点のUX問題

オーナーがページに来る理由：「自分の物件がある〇〇で、この会社は本当に使えるか」を判断するため。

現在のページで起きていること（スクロール順）：

```
Hero (固有)           ← ここは良い
よくある課題 (汎用)    ← 「どの業者サイトにも書いてある」
85%/4.9/140% (全都市同一) ← 「信頼できない数値に見える」
3つの強み (汎用)       ← 「まだ続くの？」
注意ポイント (やっと固有) ← ここまで約1,200px
市場トレンド (浅い)
サービス8項目 (汎用)
清掃 (やや固有)
田中・佐藤 (全都市同じ)  ← 「全国に同じスタッフがいるのは不自然」
ケーススタディ (全都市同一数値) ← 「やはり信頼できない」
お客様の声 (全都市同一) ← 「信用できない確信に至る」
料金 → FAQ → 問い合わせ
```

**問題の本質：** 都市固有コンテンツに到達する前に、信頼が既に損なわれている。

## Step 4：既存コンテンツの深化（🔲 未着手）

### 対象フィールドと改善方針

#### `localRules`（注意ポイント）

- **問題都市（7つ）：** karuizawa, nasu, kobe, aso, kagoshima, wakayama, takamatsu
  - 「地域に根ざした清掃・設備管理」「ファミリーゲストへの安全・清潔対応」などの汎用テンプレートが残存
- **改善基準：** 京都・ニセコ水準（具体的な地名・条例・課題を含む）
- **良い例：**
  - 「桜・紅葉・祇園祭の超繁忙期清掃体制」（京都）
  - 「スキー用具汚れ・積雪後の泥・暖房フィルター管理」（ニセコ）

#### `marketAnalysis`（市場トレンド）

- **現状：** 200〜350文字・抽象的
- **改善後：** 400〜500文字。以下を必須化：
  - 主要ゲスト国籍・層（例：台湾・香港インバウンド40%、国内ファミリー50%）
  - ピーク月と需要理由
  - 平均泊単価の目安レンジ（低シーズン・ハイシーズン）
  - 競合との差別化ポイント

#### `cleaningFeature`（清掃特徴）

- **現状：** 1〜2文（60〜85文字）のみ
- **改善後：** 3〜4文（150〜200文字）。その都市特有の清掃難題を具体的に記述
- **良い例（ニセコ）：** 「外国人スキーヤーが求めるホテル品質の清掃をニセコで実現。スキー用具汚れ・積雪後の泥・暖房フィルターまで徹底管理し、高単価レビュー獲得を支援します。ハウスキーパーは全員英語対応可能で、外国語でのチェックイン案内も代行します。」

#### `cleaningPrices`（清掃料金）

- **現状：** 34/41都市が完全に同一価格
- **改善後：** 地域の物価・需給・物件種別を反映した価格（単位：円）

|---|---|---|---|---|---|---|---|
| プレミアム | niseko, hakone, karuizawa | 6,500 | 8,500 | 12,000 | 16,000 | 22,000 | 30,000 |
| ハイ | sapporo, yokohama, osaka, fukuoka, nagoya | 5,500 | 7,200 | 10,000 | 13,500 | 18,000 | 25,000 |
| リゾート | atami, beppu, yufuin, miyakojima, ishigaki, onna | 5,500 | 7,500 | 10,500 | 14,000 | 19,000 | 26,000 |
| スタンダード+ | kyoto, kanazawa, kobe, nagasaki, matsuyama, kawaguchiko, nikko | 5,000 | 6,800 | 9,500 | 13,000 | 17,000 | 23,000 |
| スタンダード | その他 | 5,000 | 6,500 | 9,000 | 12,000 | 16,000 | 22,000 |

#### `localFaqs`（地域FAQ）

- **現状：** 3問
- **改善後：** 5〜6問。以下の5テーマをバランスよくカバー：
  1. 季節需要・繁忙期（ピーク月・価格戦略）
  2. ゲスト対応（インバウンド・多言語・文化的配慮）
  3. 清掃・物件管理（地域特有の課題）
  4. 収益・価格設定（稼働率・単価目安）
  5. 運営体制・サポート（対応スピード・緊急時対応）

> **注：** 条例・法規に関するFAQは `localRegulations` セクション（Step 5）に移管し、そちらで網羅する。

## Step 5：新規セクション3つを追加（🔲 未着手）

### 新規① `seasonalCalendar`（繁忙期カレンダー）

オーナーが最も知りたい「いつ稼げるか」を月別で視覚化する。

**cities.jsonスキーマ：**

```json
"seasonalCalendar": [
  {"month": 1, "demandLevel": 2, "note": "冬の閑散期"},
  {"month": 2, "demandLevel": 5, "note": "雪まつり・最繁忙"},
  ...（全12ヶ月）
]
```

- `demandLevel`: 1=閑散, 2=低, 3=普通, 4=高, 5=超繁忙

**HTMLでの表示：** 月ごとのバーまたはヒートマップ形式で視覚化。

---

### 新規② `marketPricing`（エリア相場データ）

具体的な数字で収益イメージを伝える。競合が手を抜きがちな差別化ポイント。

**cities.jsonスキーマ：**

```json
"marketPricing": {
  "lowSeason":  {"avgNightly": "¥8,000〜¥12,000", "occupancyPct": 40},
  "highSeason": {"avgNightly": "¥20,000〜¥35,000", "occupancyPct": 85},
  "guestProfile": "主要ゲスト層（例：台湾・香港インバウンド中心）",
  "peakPeriod": "ピーク期間（例：7〜8月の夏シーズン・年末年始）"
}
```

---

### 新規③ `localRegulations`（地域条例・法規情報）

法規情報はオーナーの最大の不安ポイント。競合が省略しがちな箇所で差別化できる。localFAQsの条例テーマもここに集約する。

**cities.jsonスキーマ：**

```json
"localRegulations": {
  "registrationType": "住宅宿泊事業法（民泊新法）",
  "maxAnnualDays": 180,
  "cityRestrictions": "市独自の制限（例：住居専用地域は週末のみ）",
  "authority": "届出先の行政窓口名",
  "keyNotes": "オーナーが特に注意すべき点"
}
```

## 確定：新しいページ構成（都市ページ）

スクロール量を現状比**約40%削減**し、固有コンテンツ密度を上げる。

```
① Hero                  ← 固有キャッチ・画像（現状維持）
② 市場概況 + 相場データ   ← marketAnalysis拡充 + marketPricing新規
③ 繁忙期カレンダー        ← seasonalCalendar 新規追加
④ 注意ポイント            ← localRules 強化
⑤ 清掃の特徴             ← cleaningFeature 拡充
⑥ 条例・法規情報          ← localRegulations 新規追加
⑦ サービス（簡略）         ← 現状の1/3に縮小、詳細はトップへリンク
⑧ 料金プラン             ← cleaningPrices 都市別価格に修正
⑨ 収益シミュレーター       ← 現状維持
⑩ よくある質問            ← localFaqs 5〜6問に拡充
⑪ 対応エリア             ← 当該都市の対応エリアのみに絞る（全国タブ削除）
⑫ お問い合わせ            ← 現状維持
⑬ Note記事              ← 現状維持
```
