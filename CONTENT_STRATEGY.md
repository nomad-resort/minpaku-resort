# Minpaku Resort 都市ページ 改善方針書

## 背景：なぜ改善が必要か

Google Search Consoleで `/aso/` `/chuo/` などの都市ページが「重複コンテンツ」と判定され、インデックス登録に失敗している。

**根本原因：** 171都市のページのうち、約94%のHTMLが全都市で同一。Googleは「これは同じページが171個ある」と判断し、自分で"正規ページ"を選び直してしまう。

---

## 現在の構成

都市ページは `index.html`（テンプレート）+ `data/cities.json`（都市データ）をビルドスクリプトで合成して生成している。

### 都市ごとに異なるコンテンツ（現状）
- ヒーロー画像・キャッチコピー
- 「○○の注意ポイント」3カード
- 市場トレンド解説（1段落）
- 清掃特徴タイトル・説明
- 地域FAQ（3問）
- 対応エリア一覧

### 全都市で共通（＝問題）のコンテンツ
- お客様の声（都市名だけ置換、写真・文章は同一）
- 実績・ケーススタディ（稼働率85%・評価4.9・収益140%UPが全都市同じ）
- スタッフ紹介（田中・佐藤が全都市に登場、同じ写真）
- 清掃料金（ニセコも阿蘇も東京も同じ価格）
- サービス紹介8項目
- 収益シミュレーター
- 運営体制・強み・実績数値
- お問い合わせフォーム

---

- **`og:url` タグを全171都市ページに追加**（OGP修正）
  - `index.html` テンプレートに追加
  - 既存の全都市ページへ一括注入済み

### Step 2：都市数を絞り込む（**方針確定**）

**171都市 → 41都市**に絞り込む。

選定基準：インバウンド客数が多い・民泊市場が活発・検索需要がある都市に集中。
薄いコンテンツで大量にページを持つより、少数精鋭で質の高いページにする。

削除した都市ページは **`robots.txt` で noindex 指定 + サイトマップから除外** する。

### Step 3：信頼破壊コンテンツの除去（都市絞り込み後に実施）

以下のセクションを**都市ページから削除する**。

| セクション | 削除理由 |
|---|---|
| `お客様の声`（voices） | 全都市で同一文章・同一写真。オーナーが複数都市を比較すると即バレする |
| `実績・ケーススタディ`（proofs） | 全都市で稼働率85%・評価4.9・収益140%という同一数値。信頼性ゼロ |
| `スタッフ紹介`（staff） | 田中・佐藤が全国に存在するのは不自然。地域密着の嘘になっている |
| `運営実績数値`（numbers） | 「対応件数・稼働率向上・満足度」が全都市同一 |
| `3つの強み`（strengths） | 全都市共通の汎用テキスト |

→ これらは**トップページ（index.html）のみ**に残す。都市ページからはトップへの誘導リンクを置く。

### Step 4：既存の都市固有コンテンツを深化させる

絞り込んだ約40都市について、すでにある都市固有セクションの**質を引き上げる**。

#### `localRules`（注意ポイント）
- 現状：都市によって品質にムラあり（京都・ニセコは具体的、阿蘇は汎用的）
- 改善後：全都市を京都・ニセコ水準に統一。具体的な条例名・制限日数まで踏み込む

#### `marketAnalysis`（市場トレンド）
- 現状：1段落のテキスト
- 改善後：「ゲスト国籍比率」「繁忙月と単価目安」「競合との差別化ポイント」の3ブロックに拡充

#### `cleaningFeature`（清掃特徴）
- 現状：都市固有だが内容が薄い都市もある
- 改善後：その都市特有の清掃難題（ニセコ=凍結・スキー用具汚れ、京都=古建材、沖縄=塩害・湿気）を具体的に記述

#### `cleaningPrices`（清掃料金）
- 現状：大半の都市で同一価格
- 改善後：地域の物価・需給を反映した価格設定（東京・ニセコ等は+20〜30%、地方は標準）

#### `localFaqs`（地域FAQ）
- 現状：3問、質は良い
- 改善後：5〜6問に拡充。条例・季節・ゲスト対応の3テーマをカバー

### Step 5：新規セクションを追加する

以下の3セクションを新規追加する。オーナーが「この代行業者は本当にこのエリアを知っている」と感じるための情報。

#### 新規①：繁忙期カレンダー
```
京都の例：
3月下旬〜4月上旬（桜）→ 需要+60%、稼働率95%超
7月（祇園祭）→ 需要+30%
10〜11月（紅葉）→ 需要+50%
1〜2月（オフシーズン）→ 価格調整と長期滞在需要の取り込みが重要
```
オーナーが最も知りたい「いつ稼げるか」を視覚的に伝える。

#### 新規②：地域条例・法規情報（詳細）
```
京都市の例：
・住居専用地域：年間営業180日以下（全国共通の住宅宿泊事業法に加え市独自制限）
・特定エリア（祇園周辺等）：さらに厳しい制限の可能性あり
・近隣住民への事前説明義務
・苦情対応窓口の設置義務
```
法規情報は競合が手を抜きがちで、かつオーナーの最大の不安ポイント。

#### 新規③：エリア相場データ（Airbnb平均単価・稼働率目安）
```
ニセコの例：
冬季（12〜3月）平均泊単価：¥35,000〜¥80,000
夏季（7〜8月）平均泊単価：¥15,000〜¥25,000
年間稼働率目安：冬70% / 夏40%
```
抽象的な「稼働率最大化」ではなく、具体的な数字で収益イメージを伝える。
